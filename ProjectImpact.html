3D Game Engine (WIP)

When I started this project, I knew I wanted to learn how to work with DirectX11 without using the Effects system, but I didn't really know where to begin.
As a guiding hand I looked at other projects and their arcitecture, which helped me get started. I learned a lot from Microsofts Documentation on Win32, as well as looking into how others had solved the issues I ran into. I found a render engine called "hw3d" by PlanetChili, after examining it, I saw things I liked, and things I didn't like as much. I took inspiration from the things I liked, and came up with my own solutions for what I didn't like.
As an example of what I didn't like, the hard-coded order of objects in the apps render function(left).
Where my solution is a layered structure, where Objects to be rendered get's handled elsewhere(right).

The idea of the layer structure, is to separate each aspect into its own domain. With this in mind, I made an event handler that sends events through the layers like a sift to get handled.

First register the event which should be called, and what should triger it.


    • Then when the trigger happenes, we send the event through.

    The Rendering & Update loop happens in the reverse order Game > UI > Debug > Application.
    The engines objects are built up from components, currently using a very simple component system which for now it's engouh.
    The engine can currently do just a few things render primitive objects, diffuse texture rendering, point & directional lighting, dynamic camera movement
    Next on my list of things to implement is the option to import assets, multiple texture maps such as hight-maps, normal maps and rooughness maps etc.
